┌─────────────────────────────────────────────────────────────────────────────┐
│ 2. RING_BUFFER.HPP - Lock-Free SPSC Communication                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                               │
│  PERFORMANCE CHARACTERISTICS:                                                │
│  • Latency: 50-100 nanoseconds per operation                                 │
│  • Throughput: 15-20 million operations per second                          │
│  • Memory: Minimal overhead (just head/tail pointers)                       │
│  • Guarantees: Wait-free, lock-free, deterministic                          │
│                                                                               │
│  DESIGN PRINCIPLES:                                                          │
│  • Single Producer, Single Consumer (optimized for dedicated threads)        │
│  • Lock-Free: Uses only atomic operations                                    │
│  • Cache-Friendly: Producer head and consumer tail on separate cache lines  │
│  • Memory Ordered: Acquire/release semantics for correctness               │
│  • No Allocations: All memory pre-allocated                                  │
│                                                                               │
│  FEATURES:                                                                   │
│  • RingBuffer<T, SIZE> - Core SPSC ring buffer                             │
│    ├─ enqueue(T&) - Lock-free insertion                                    │
│    ├─ dequeue(T&) - Lock-free removal                                      │
│    ├─ enqueue_batch() / dequeue_batch()                                    │
│    ├─ peek() - Non-destructive read                                         │
│    └─ size(), available() - State queries                                   │
│                                                                               │
│  • SPSCQueue<T, SIZE> - High-level queue with typed API                    │
│    ├─ Producer interface - Type-safe producer-only access                  │
│    ├─ Consumer interface - Type-safe consumer-only access                  │
│    └─ Explicit get_producer() / get_consumer()                             │
│                                                                               │
│  MEMORY LAYOUT:                                                              │
│  ┌─────────────────────────────────────┐                                   │
│  │ Producer Head (atomic<uint32_t>)    │  64-byte cache line 1             │
│  ├─────────────────────────────────────┤                                   │
│  │ Consumer Tail (atomic<uint32_t>)    │  64-byte cache line 2             │
│  ├─────────────────────────────────────┤                                   │
│  │ Data Array [0..SIZE-1]              │  Circular buffer storage          │
│  └─────────────────────────────────────┘                                   │
│                                                                               │
│  SIZE must be power-of-2 for efficient bitmasking (modulo operation)        │
│                                                                               │
└─────────────────────────────────────────────────────────────────────────────┘

│                                                                               │
│  FEATURES:                                                                   │
